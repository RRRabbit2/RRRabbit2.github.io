    # 布隆过滤器 Bloom Filter

    # 什么是布隆过滤器？

    1970年由布隆提出。

    由一个很长的二进制向量和一系列随意映射函数组成。

    是一种基于概率的数据结构，主要用来判断某个元素是否在一个集合中，具有运行速度快，占用内存小的优点，具有一定的误判率，删除困难。

    误判率： 可以保证一个元素一定不在集合内，不能保证一个元素一定在一个集合内，只能保证可能在集合内。

    # 为什么需要布隆过滤器？

    判断一个元素是否在一个集合中：网络爬虫时，判断一个网页是否被爬过；判断一个邮件地址是否在黑名单中；判断英文单词是否拼写正确。

    1. 将所有的元素都保存在计算机中——链表O(n)、树O(longn)、哈希表O(n/k)，随着数量增加，需要的元素空间越来越大，检索速度变慢。为保证哈希表的查找效率，其存储效只有50%。

      Hash Table 常用于频繁进行 key/value 模式的查找中。(查找模式，如匹配查找) 哈希表最大的优点在于查找速度快，但存储时可能发生collision(冲突)。 哈希表大多使用open addressing来解决collision，此时search的时间复杂度计算公式为： 1/( 1 - n/m ) 其中，n与m分别表示存储的记录数与哈希表的长度，即装填因子( load factor ) 故，若哈希表半满，即 n/m >= 1/2，则每次的search次数可能会 >= 2 因此，为了保证Hash Table在 key/value 查找模式中的优势，一般，其存储效率不会超过50%。

    2. 布隆过滤器，只需要哈希表1/4 或 1/8的大小。

      why? ⚠️

    上图中，m=18，k=3，h1、h2 和 h3 是三个哈希函数将输入值分别映射成比特向量上的某个位置上。

    插入 x 值（对应红线），则将经过 h1、h2 和 h3 三个哈希函数映射到的比特向量上的值改为 1。

    y（对应绿线）和 z（对应蓝线）值也一样操作。

    如果要查找 x、y、z，通过布隆过滤器可以找到对应的比特向量都为 1，我们只能确定想 x、y、z 可能在集合中。这里我们我上图中，可以发现，x 和 z 在 h2 哈希函数映射后映射到的是同一个比特向量位置（第 6 个），同时 y 和 z 在经过 h3 映射后也是同一个比特向量位置（第 12 个）。也就是说，当我们通过布隆过滤器判断一个元素是否在集合中时，即使对应的比特向量位置全为 1，也可能是其他元素映射后导致的。如果要查找 w 值，经过哈希映射后，可以发现有一个比特向量上的值不为 1，那就可以判断，w 值一定不在集合中。

    由于不同的元素经过不同哈希函数的映射可能会映射到相同的比特空间中，所以在检查元素是否存在时可能会发生误判。插入的元素越多，发生误判的概率越大。

    误报率和数据集的容量n，哈希函数个数K，总比特数m相关，当布隆过滤器使用率为50%时，误报率最小。

    # 布隆过滤器在比特币中的应用

    比特币钱包如何计算一个账户的所有UTXO

    1. 下载所有的账本——保护隐私，浪费空间和贷款
    2. 告诉全节点自己的地址——泄露隐私
    3. 告诉全节点部分自己钱包的信息，全节点返回所有可能的UTXO——节点构造一个布隆过滤器
