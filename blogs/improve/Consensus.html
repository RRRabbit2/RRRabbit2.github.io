    # 共识

    共识算法是区块链技术的核心，但是学了一年还是觉得对共识算法的了解不够深入，周末，无心做事，尽可能全面地了解一下共识算法吧。

    # 1. PoW(Proof of Work, 工作量证明)

    • 09年比特币诞生以来，被广泛测试和使用
    • 有竞争共识，耗能巨大，环境不友好，易受规模经济的影响？（51%攻击）
    • 应用在Bitcoin，Ethereum，LiteCoin，Dogecoin中（后两者没有听说过）。
    • 矿工要想在区块链中添加一个区块，必须解决某种“复杂但是无用”问题（通过穷举运算，找到一个随机的Nonce值，使得区块的哈希以n个0开头）。

    # PoS（Proof of Stake，权益证明）

    • 节能，攻击代价大，不易受规模经济的影响。
    • 有竞争共识，无利害关系攻击问题？
    • 应用在Ethereum（正在从PoW向PoS过渡），Peercoin，Nxt中。
    • 随机（伪随机，币龄越多被选中的概率越大）选择一个矿工来打包将交易（出块），出块成功的矿工将收到与币龄呈正相关的收益。挖矿成功后，币龄清零。

    # DPoS（Delegated Proof of Stake，委任权益证明）

    • 节能、快速，EOS的块时间为0.5秒
    • 协同型共识，略微中心化，拥有高权益的参与者可投票使自己成为一名验证者。
    • 应用在BitShares、Steemit、EOS、Lisk、Ark中。
    • 权益持有者可以从许多候选人中选出一批见证人，这些见证人以轮询的方式产生区块，一次产生一个区块，防止一个节点产生连续的块，如果在规定时间内不能产生区块，则跳过该见证人，若见证人多次在规定时间内未产生区块，权益持有者通过投票使其退出，并选举出新的见证人。

    # PBFT（Practical Byzantine Fault Tolerance，实用拜占庭容错）

    • 高速、可扩展
    • 通常用于私有网络和许可网络
    • 应用于Hyperledger Fabric、Stellar、Ripple、Dispatch中。
    • PBFT、FBA（Federated Byzantine Agreement，联邦拜占庭协议），FBA吞吐量高，交易开销低，网络扩展性强，FBA共识算法是目前提出的最好的分布式共识发现算法。

    # RAFT（分布式一致性协议）

    • 比Paxos更简单，但是具有同等的安全性，多种语言可实现。
    • 用于私有网络和许可网络
    • 应用于IPFS Private Cluster、Quorum中
    • 对拜占庭将军问题进行简化，假设将军中没有叛军，信使可靠但是可能被暗杀。在所有的将军中选出一个大将军，所有的决定由大将军来做。有Leader、Candidate和Follower三种角色。Candidate需要收到大于n/2的节点的投票才能成为Leader。

    # DAG（Directed Acyclic Graphs，有向无环图）

    • 非线性结构、高度可扩展（Gossip中，增加节点的数量，让所有节点都收到消息所需要发送的轮次，并不会产生巨大的变化）、快速、节能、立即实现终结性（Finality）。
    • 只能通过Oracle实现智能合约
    • 应用在Iota、HashGraph、Byteball、RaiBlocks/Nano中
    • 一个新的区块的父区块可能有多个，一个区块只需要得到前面部分区块的认可就可以并行地添加到区块中，而不是只能串行地只能添加一个区块。
    • 例子Gossip，【Gossip参考】图解Gossip：可能是最有趣的一致性协议 - 简书 (jianshu.com) (opens new window)

    # PoC（Proof of Capacity，容量证明）

    • 将候选者的磁盘空间作为竞争的标准。候选者通过分配一定数量的内存或磁盘空间用于解决提供服务者所提供挑战的方式，显示其对某个服务的合法兴趣。

    # 课程，共识部分

    分布式系统分为同步和异步分布式系统

    （阻塞和非阻塞）

    同步：请求执行完成后再响应（打电话）

    异步：先响应了，执行不受影响后续执行。（发短信）异步系统中对节点运行速度和消息传递的延迟都没有限制。

    区块链系统一般采用异步（弱同步）分布式系统。

    异步场景下，超时并不能表明节点没有响应（可能是返回的消息正在路上或者事务正在被处理）。

    执行的一致性和输入的一致性

    随机故障：BFT，拜占庭故障

    遗漏故障：CFT，非拜占庭故障
