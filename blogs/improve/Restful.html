    # Ubuntu16上安装Restful的C++库

    [TOC]

    # 0 概述

    “REST是一组架构约束条件和原则”，如果一个架构符合REST的约束条件和原则，就称它为RESTful架构。

    REST受Web的影响很深，理论上REST架构风格不是绑定在HTTP上的，但是目前HTTP是唯一于REST相关的实例。

    # 性能对比

    GitHub上有很多用C++写的Restful接口，有人对star数量较多的C++ Restful库进行了对比，第二列和第三列是性能指标，第四列体现了易用性。

    Restful库 98%请求中的最大时间（越小越好） 每秒平均处理的请求数（越大越好） Sample代码的行数
    C++ : cpprestsdk / default JSON implementation 51 30.70 48
    C++ : cpprestsdk / RapidJSON 44 47.06 47
    C++ : restbed 7 224.18 39
    C++ : pistache 6 319.99 40
    PHP : Native implementation 10 146.95 14

    其中，微软的cpprestsdk在GitHub上的star数量最多；pistache的性能最好。

    # 安装难易对比

    个人安装测试体验对比：

    Restful库 安装难度 是否需要手动配置依赖 官方文档明确程度 相关非官方文档数量 所依赖的库安装难度
    C++：cpprestsdk 简单 否 好 多 简单
    C++ : restbed 中等 是 好 少（但有用） 中等
    C++ : pistache 困难(安不上) 未知 跟着官方文档走，安不上 少 困难

    就安装体验而言，微软的cpprestsdk是最容易安装的，文档也做的比较好，能够很快安装好，但是Ubuntu16上运行有段错误。性能测评中pistache有优势，但是它依赖的库安装起来有难度，官方文档看上去很清楚，但是执行起来有很多问题，未能成功安装。restbed各项指标都不算突出，按照官方文档走，会遇到一些问题，但基本能通过参考其他非官方文档解决，安装成功。

    安装测试从pistache开始，失败之后又试了微软的cpprestsdk，又失败之后试了restbed，restbed成功安装并可以跑通测试用例，这里对restbed的安装过程进行记录，最后记录一下前面两种的问题所在。

    # 1 版本信息

    • Ubuntu16.04
    • GCC 5.4.0，支持 c++14

    # 2 restbed 4.7

    经过试验在Ubuntu16.04上，restbed能够成功安装并使用，目前没有快速安装方法。

    restbed4.7 依赖的库：cmake，asio（版本必须为：1.12.0），openssl，catch2。

    4.7版本的restbed：https://github.com/corvusoft/restbed/tree/4.7

    # 拉取代码

    有两种拉取方法，一种是递归地拉取，会将以来的包也拉取下来，速度较慢，且依赖的包可能拉取不成功；另一种是非递归拉取，仅拉取restbed这一层代码，不拉取依赖的包，较快，但是依赖包要自行安装。

    # 递归的拉取

    递归拉取restbed源代码，会将依赖的包全部拉取下来，存在 /restbed/dependency 下。

    git clone --recursive https://github.com/corvusoft/restbed.git
    1

    # 非递归拉取

    非递归拉取下来后，进入 /restbed/dependency 目录，按需要根据 GitHub 上对应的路径拉取依赖库。

    git clone https://github.com/corvusoft/restbed.git
    1

    这样拉取下来，所在的分支是master，需要切换到4.7分支。（比官方的教程多一步）

    cd restbed
git checkout 4.7
    1
    2

    # 检查依赖库版本

    这里采用递归拉取的方法，拉取成功，并切换到4.7版本后，进入到 /restbed/dependency/asio 目录执行以下命令。

    git checkout asio-1-12-0
    1

    切换依赖库asio到1.12.0版本，如果不切换安装时会出现错误。

    其他两个依赖库没有版本要求，由于欧冶依赖的openssl1.0.2g版本，注意不要全局安装其他版本的openssl。

    # 安装

    在 /restbed 目录下，执行以下命令。

    mkdir build
cd build
cmake [-DBUILD_SSL=NO] [-DBUILD_TESTS=NO] ..
make install
make test
    1
    2
    3
    4
    5

    camke 命令用于检查依赖包是否都已经存在，它只会检查是否存在，不会检查版本，如果asio版本不对，此处不会报错。如果报错，根据报错的内容，对照 /camke目录中cmake文件的要求，检查对应的依赖库是否已经安装在系统中，或者是否已经在 /restbed/dependency 中。

    用zsh执行cmake的时候可能会出现（只有zsh会出现）：

    解决方法：

    vi ~/.zshrc
    1

    编辑，新增行

    setopt no_nomatch
    1

    重新执行该配置脚本

    source ~/.zshrc
    1

    make install 命令安装 restbed，刚开始make（执行不到1%），大概率是saio版本的问题。

    make test 命令进行测试，一共有115个点，全部通过后应该会输出以下信息。

    # 配置

    make install 出来的include和lib在 /restbed/distribution 目录下，将 /include 中的内容拷贝到 /usr/include 中，在 /restbed/distribution/inlcude 目录下，执行以下命令即可。

    sudo cp -r * /usr/include
    1

    将 /restbed/distribution 目录下的 /library 中的 librestbed.a 文件复制到应用程序的目录中。

    # 使用

    目录结构如下：

    其中，hello为需要显示的内容，librestbed.a 文件为 restbed 的静态链接库文件，test为编译生成的可执行文件 ，test.cpp 为测试的源代码。

    test.cpp 中的测试代码如下：

    #include <string>
#include <memory>
#include <cstdlib>
#include <fstream>
#include <restbed>
#include <streambuf>
#include <iostream>

using namespace std;
using namespace restbed;

void get_method_handler(const shared_ptr<Session> session)
{
    const auto request = session->get_request();
    const string filename = request->get_path_parameter("hello");
    ifstream stream("./" + filename, ifstream::in);

    if (stream.is_open())
    {
        ifstream in("hello");
        istreambuf_iterator<char> begin(in);
        istreambuf_iterator<char> end;
        string some_str(begin, end);
        const multimap<string, string> headers{
            {"Content-Type", "text/html"},
            {"Content-Length", ::to_string(some_str.length())}};
        session->close(OK, some_str, headers);
    }
    else
    {
        session->close(NOT_FOUND);
    }
}

int main(const int, const char **)
{
    auto resource = make_shared<Resource>();
    resource->set_path("/static/{filename: [a-z]*\\.html}");
    resource->set_method_handler("GET", get_method_handler);

    auto settings = make_shared<Settings>();
    settings->set_port(1984);
    settings->set_default_header("Connection", "close");

    Service service;
    service.publish(resource);
    service.start(settings);

    return EXIT_SUCCESS;
}

    # 3 cpprestsdk

    它使用起来确实很方便，但是：

    在Ubuntu16上，cpprestsdk可以安装成功，测试案例也可以编译成功，甚至也可以成功获取到网页信息，但是，会报段错误，并且定位不到错误位置，两台Ubuntu16的机器上都是如此，但是Ubuntu20上可以正常使用，考虑使用成本，放弃它。（官方文档中说明支持Ubuntu16.04及以上版本系统）

    以下是失败的安装过程。

    使用apt-get快速安装，不需要考虑依赖问题。

    # 快速安装

    sudo apt-get install libcpprest-dev
    1

    用以下命令检查是否安装成功：

    dpkg -l | grep restbed
    1

    # 使用

    测试源代码：

    /* rest_test.cpp */
#include <cpprest/http_client.h>
#include <cpprest/filestream.h>

using namespace utility;                    // Common utilities like string conversions
using namespace web;                        // Common features like URIs.
using namespace web::http;                  // Common HTTP functionality
using namespace web::http::client;          // HTTP client features
using namespace concurrency::streams;       // Asynchronous streams

int main(int argc, char* argv[])
{
    auto fileStream = std::make_shared<ostream>();

    // Open stream to output file.
    pplx::task<void> requestTask = fstream::open_ostream(U("results.html")).then([=](ostream outFile)
    {
        *fileStream = outFile;

        // Create http_client to send the request.
        http_client client(U("http://www.bing.com/"));

        // Build request URI and start the request.
        uri_builder builder(U("/search"));
        builder.append_query(U("q"), U("cpprestsdk github"));
        return client.request(methods::GET, builder.to_string());
    })

    // Handle response headers arriving.
    .then([=](http_response response)
    {
        printf("Received response status code:%u\n", response.status_code());

        // Write response body into the file.
        return response.body().read_to_end(fileStream->streambuf());
    })

    // Close the file stream.
    .then([=](size_t)
    {
        return fileStream->close();
    });

    // Wait for all the outstanding I/O to complete and handle any exceptions
    try
    {
        requestTask.wait();
    }
    catch (const std::exception &e)
    {
        printf("Error exception:%s\n", e.what());
    }

    return 0;
}
