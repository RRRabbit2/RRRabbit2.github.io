    # 学习Flask

    本机环境：win10，python3.7，powershell

    # 安装

    官方推荐使用最新版本的 Python 。 Flask 支持 Python 3.6 以上版本。

    如果要支持 async ，因为会用到 contextvars.ContextVar ，所以需要 Python 3.7 以上版本。

    用python --version查看本地的python版本，目前版本是3.7，不需要升级。

    建议用虚拟环境来进行开发，这里我有点疑惑。官方给出了说明：

    为什么要使用虚拟环境？随着你的 Python 项目越来越多，你会发现不同的项目 会需要不同的版本的 Python 库。同一个 Python 库的不同版本可能不兼容。

    虚拟环境可以为每一个项目安装独立的 Python 库，这样就可以隔离不同项目之 间的 Python 库，也可以隔离项目与操作系统之间的 Python 库。

    Python 内置了用于创建虚拟环境的 venv (opens new window) 模块。

    这个解释说服了我，很有道理hhhh🤭

    venv内部的原理以及其他虚拟环境容器（存疑），如之前遇到的docker，还未深入了解，在此埋坑🕳

    首先新建项目文件夹myproject，进入这个文件夹中，用指令py -3 -m venv venv创建一个虚拟环境。创建完成后，项目文件夹中会有一个venv文件夹。

    > mkdir myproject
> cd myproject
> py -3 -m venv venv

    输入指令venv\Scripts\activate激活虚拟环境，这个指令大意为：执行项目目录下\venv\Scripts\activate文件，每个venv文件夹中都有，刚开始理解错误，在此mark🐎一下，激活虚拟环境后，命令行会显示虚拟环境名称。

    在此虚拟环境中用指令pip install Flask安装Flask。

    # 简单程序

    from flask import Flask  #导入Flask类

app = Flask(__name__)  #创建Flask类实例

@app.route("/")  #告知Flask触发函数的URL
def hello_world():
    return "<p>Hello, World!</p>"  #默认返回内容为HTML格式

    保存文件，不要命名成flask.py，可以命名成app.py或者wsgi.py，保存为后二者不需要导出FLASK_APP环境变量。

    powershell中导出环境变量并运行。

    > $env:FLASK_APP = "hello"
> flask run 或 python -m flask
    1
    2

    网页运行在 http://127.0.0.1:5000/ 上。

    如果想让同一个网络中的机器通过IP访问本机的网页，可以将指令flask run换成flask run --host=0.0.0.0，公开监听所有公开的IP。

    打开调试模式：flask run之前执行指令$env:FLASK_ENV = "development"，服务器会在修改代码后自动重启，如果出错也会有交互式调试器。
